# Provider Contract

The canonical contract lives in `packages/backend-contracts/provider.ts`.

Important rules:

- Domain IDs are generated by application code.
- Provider-native IDs never leave an adapter.
- Mutable aggregates use `revision`.
- Revision-sensitive writes accept `expectedRevision`.
- Important mutations accept an idempotency key.
- Cursors are opaque and returned through `Page<T>`.
- Atomic behavior is exposed through named domain commands, not generic transaction callbacks.

The first executable implementation is `packages/provider-local/adapter.ts`.
